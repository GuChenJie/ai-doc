# RAG 知识整理

这里整理的是 RAG 的 chunk 相关知识，重点放在“怎么切、切成什么样、什么时候选哪种切分方式”。

## chunk 的分类

### 1. 基础切分

适合做最稳妥的通用起点，优先保证实现简单、结果稳定。

- 固定长度切分
- 递归切分
- 基于句子的切分

### 2. 结构切分

适合保留原文结构，让 chunk 更接近文档本身的层次。

- Markdown 切分
- 对话式切分
- 章节 / 标题感知切分

### 3. 语义切分

适合更关注内容主题，而不是纯长度。

- 语义切分
- 主题切分
- 基于相似度的段落合并

### 4. 高级切分

适合复杂文档或者更高质量的检索场景。

- Parent-Child chunk
- Agentic chunk
- 自适应切分

### 5. 混合切分

适合把前面几类方法组合起来，兼顾质量和覆盖率。

- 结构 + 固定长度
- 语义 + 固定长度
- 结构 + 语义
- 自适应混合策略

## 选型建议

- 文档结构清晰时，先用结构切分。
- 长文档且主题变化明显时，优先看语义切分。
- 如果要兼顾召回和稳定，混合切分通常更合适。
- 如果只是先把链路跑通，固定长度切分最容易落地。
